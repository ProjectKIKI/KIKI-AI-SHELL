# lsh - LLM Shell

자연어로 명령을 설명하면, LLM이 적절한 bash 명령을 제안해주고 사용자가 확인한 뒤 실제로 실행하는 **비서형 쉘**입니다.
'/bin/lsh'를 로그인 쉘로 설정하면, 사용자가 SSH/콘솔 로그인 시 곧바로 LLM 보조 쉘 환경으로 진입할 수 있습니다.

이 쉘의 주요 목적은 LLM으로 어느정도 수준까지 작업을 실시간으로 반영 및 적용이 가능한지 확인하는 위함 입니다.

> 이 버전은 **Python 기반 프로토타입**입니다.  
> 추후 Go로 이식하거나 기능을 확장하기 위한 베이스 코드입니다.
> 어디까지 엘엘엠이 들어올수 있는 거에요~~

## 특징

- **자연어 → 쉘 명령 추천**
  - → `"/var/log/messages에서 error 나는 마지막 50줄만 보고 싶어"`  
  - → `grep -i "error" /var/log/messages | tail -n 50`
- **실행 전 항상 PLAN 표시 및 사용자 확인**
- **위험 패턴 필터링**
  - `rm -rf /`, `mkfs`, `dd`, `shutdown` 등 일부 위험 명령은 추가로 경고
- **컬러 출력**
  - 프롬프트, PLAN, 명령 목록을 색상으로 구분 (연두/청록 계열)
  - 추후 색깔 변경 가능하도록 수정 예정
- **bash 폴백**
  - LLM 호출 실패 시, 원하면 바로 `/bin/bash`로 진입
  - `bash` 명령 입력으로 언제든 대화형 bash로 진입 가능
  - 과연 다른 쉘이 필요한가?
- **비대화식 상황 자동 처리**
  - TTY가 아닐 경우 (예: scp, cron) 자동으로 `/bin/bash`로 폴백
  - 자체 TTY 구성이 필요한가?
